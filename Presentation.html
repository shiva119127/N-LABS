Solution & Live Demo · 1:00 – 2:00

Here's How We Built It ~60 sec

"Our project has three layers that each one of us owns:"

🎨 Frontend (HTML + CSS)
Dark glassmorphism design, CSS variables for theming, responsive grid, scroll reveal animations, and a brand-new light/dark mode toggle that remembers your preference.
⚡ Interaction Layer (script.js)
15 independent features: magnetic buttons, 3D card tilt, custom cursor, scroll progress bar, typing effect, active nav highlighting, and a back-to-top button.
🌐 3D Background (main.js)
Three.js scene with floating particles, a wireframe icosahedron, and a torus knot. They rotate, breathe, and fade out on scroll — powered by GSAP for silky animation.
📊 Data Dashboard
Real Chart.js graphs showing revenue and project metrics. KPI bars animate in on scroll. Counter numbers count up from zero. Makes the page feel alive with real data.

Demo moment: Move your mouse over a service card (shows tilt + glare). Click a button (shows ripple). Hover the nav (shows active state). Scroll down 50% (back-to-top appears).

Quick Q&A Reference — if judges ask

"Why Three.js and not a video background?"
Video files are heavy (5–20MB). Our 3D canvas is ~0 bytes of assets — everything is generated in JavaScript at runtime. It also responds to mouse and scroll.
"How is this different from a template?"
Every pixel is hand-coded. No Bootstrap, no Tailwind, no page builders. Pure CSS variables, custom grid, and original JavaScript features.
"What's the performance like?"
We cap pixel ratio at 2×, use requestAnimationFrame for all animations, passive scroll listeners, and IntersectionObserver instead of scroll event spam.
"Is it accessible?"
Yes — ARIA labels on interactive elements, Escape key for menus, prefers-reduced-motion support that disables all animations for users who need it.
